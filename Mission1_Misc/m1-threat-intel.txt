3) Pattern IOC: URI path "/MyUserName_DEVICEHOSTNAME"

Mapping to This Lab (How to Frame It for Students)

- In this TechFlow MSP scenario, treat the lab’s “suspicious C2 IP” as ANALOGOUS to the published ScreenConnect IOCs above:
  - Same class of activity (post-exploitation C2 / tooling download after CVE-2024-1709 abuse).
  - Similar tradecraft: initial ScreenConnect compromise → malicious extension / payload → outbound HTTPS to attacker infrastructure.

- Recommended student activities:
  - Use the IP from EDR / NetFlow logs in this lab PLUS the published IPs above to:
    - Practice VT / urlscan / Shodan pivoting.
    - Compare infrastructure characteristics (ports, certs, hosting providers).
  - Discuss:
    - How quickly such infrastructure changes (ephemeral C2 vs longer-lived infra).
    - Why relying only on static IPs is fragile, and why pattern-based detection (URI paths, process chains, extension file names) matters.
